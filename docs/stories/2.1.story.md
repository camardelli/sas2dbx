# Story 2.1: Análise de Dependências — autoexec.sas + Dependências Implícitas

**Status:** ToDo

## Objetivo
Estender o `dependency.py` para analisar `autoexec.sas` (configurações globais de LIBNAME e macros) e suportar declaração manual de dependências implícitas via `libnames.yaml` — garantindo que o grafo de dependências capture relações que não aparecem explicitamente no código dos jobs.

## Contexto
Jobs SAS frequentemente dependem de datasets criados por outros jobs via convenção de nome ou via configurações definidas em `autoexec.sas` (LIBNAMEs globais, macros corporativas). Sem analisar o autoexec, o grafo de dependências fica incompleto, gerando notebooks com referências quebradas ou ordem de execução incorreta no workflow YAML.

## Escopo
### Incluído
- Parsing de `autoexec.sas` no `dependency.py`:
  - Extrair declarações `LIBNAME` globais
  - Extrair `%INCLUDE` de arquivos de macro
  - Adicionar ao contexto global de dependências (sobrepõe libnames locais)
- Suporte a dependências manuais em `knowledge/custom/libnames.yaml`:
  ```yaml
  # Formato estendido
  SASDATA:
    catalog: "main"
    schema: "raw"
    depends_on_jobs: ["job_001_carga_clientes"]  # NOVO: dependência explícita
  ```
- Detecção de dependências implícitas via padrão de nome de dataset:
  - Se `job_A` cria `WORK.clientes` e `job_B` lê `WORK.clientes` → dependência inferida
- `DependencyGraph` deve expor método `get_implicit_dependencies()` para inspeção
- Warning no relatório quando dependência implícita é detectada (requer validação humana)

### Excluído
- Resolução automática de macros corporativas complexas (apenas `%INCLUDE` simples)
- Análise de dependências de runtime (ex: tabelas criadas dinamicamente via `CALL EXECUTE`)
