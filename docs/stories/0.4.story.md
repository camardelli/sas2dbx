# Story 0.4: Knowledge Store — CLI + Validate + Manifest + Cache

**Status:** Done
**Prioridade:** P0 (completa o Sprint 0 antes do Sprint 1)
**Esforço:** M
**Responsável:** @dev
**Epic:** Sprint 0 — Knowledge Store Foundation
**Origem:** Code Review Sprint 0 — G3, G4, G5, G6 + Cache improvement

## Objetivo
Completar a infraestrutura do Knowledge Store com: CLI Typer+Rich operacional para os comandos de knowledge, validação de integridade/cobertura, geração automática de manifest.yaml e cache em memória no store.

## Contexto
Após as Stories 0.1-0.3, o Knowledge Store tinha infraestrutura e dados mas sem interface de usuário, sem validação e sem performance adequada para uso repetido pelo transpiler.

## Escopo
### Incluído
- `sas2dbx/cli.py` — CLI Typer + Rich com `knowledge harvest`, `knowledge build-mappings`, `knowledge validate`
- `sas2dbx/knowledge/validate.py` — `validate_knowledge_store()` com 7 checks + `ValidationReport` dataclass
- `sas2dbx/models/sas_ast.py` — adicionar `ValidationReport` dataclass
- `sas2dbx/knowledge/populate/normalizer.py` — gerar `manifest.yaml` automaticamente após `build_mappings()`
- `sas2dbx/knowledge/store.py` — lazy loading com cache em memória (`_cache` + `invalidate_cache()`)
- `pyproject.toml` — adicionar `typer[all]>=0.12` e `rich>=13.0`
- Testes: `tests/test_cli.py`, `tests/knowledge/test_validate.py`, TestManifest em test_normalizer.py, TestMappingCache em test_store.py

### Excluído
- `sas2dbx migrate` e outros comandos do pipeline principal (Sprint 1+)
- `--resume` funcional (stub apenas — Story 3.1)
