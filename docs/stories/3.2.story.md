# Story 3.2: LLM Client — Abstração de Provider (Anthropic Direct vs Khon Gateway)

**Status:** Done
**Prioridade:** P1
**Esforço:** S
**Responsável:** @dev
**Epic:** Sprint 3 — Transpilation Core
**Origem:** Revisão arquitetural @architect (R9)

## Objetivo
Implementar `transpile/llm/client.py` com abstração completa de provider — mesma interface para Anthropic direto e Khon.ai gateway — com fallback automático e configuração via `sas2dbx.yaml`.

## Contexto
O `sas2dbx.yaml` menciona `provider: anthropic | khon` mas sem detalhes de implementação. Sem abstração, qualquer mudança de provider exige modificar o engine. A abstração garante que o transpiler seja agnóstico ao provider e que o fallback (gateway indisponível → Anthropic direto) seja transparente.

## Escopo
### Incluído
- Interface base `LLMProvider` (ABC):
  ```python
  class LLMProvider(ABC):
      @abstractmethod
      async def complete(self, prompt: str, max_tokens: int, temperature: float) -> LLMResponse: ...
      @abstractmethod
      def is_available(self) -> bool: ...
  ```
- `AnthropicProvider` — wrapper direto do SDK `anthropic`
- `KhonGatewayProvider` — wrapper HTTP para gateway Khon.ai (Bearer token auth)
- `LLMClient` com seleção automática de provider:
  ```python
  class LLMClient:
      def __init__(self, config: LLMConfig):
          # tenta khon se configurado, fallback para anthropic
      async def complete(self, prompt: str) -> LLMResponse: ...
  ```
- Retry com exponential backoff (3 tentativas, configurável)
- Log de qual provider foi usado em cada call
- `LLMResponse` dataclass: `content`, `provider_used`, `tokens_used`, `latency_ms`
- Testes com mock dos providers

### Excluído
- Streaming de resposta (não necessário para code generation)
- Cache de respostas LLM

## Critérios de Aceitação
- [x] AC-1: `LLMClient` usa `KhonGatewayProvider` quando `provider: khon` configurado e gateway disponível
- [x] AC-2: Fallback automático para `AnthropicProvider` quando Khon gateway retorna erro 5xx
- [x] AC-3: Retry com backoff em falhas transitórias (rate limit, timeout)
- [x] AC-4: `LLMResponse.provider_used` indica qual provider respondeu
- [x] AC-5: `temperature: 0.0` é o default (determinístico para code gen)
- [x] AC-6: Testes com mock cobrem: anthropic direto, khon fallback, retry após falha
