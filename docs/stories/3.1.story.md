# Story 3.1: Engine de Transpilação — Checkpointing e Resume

**Status:** Done
**Prioridade:** P1
**Esforço:** M
**Responsável:** @dev
**Epic:** Sprint 3 — Transpilation Core
**Origem:** Revisão arquitetural @architect (R3)

## Objetivo
Adicionar checkpointing ao `transpile/engine.py` via `migration_state.json`, permitindo retomar (`--resume`) uma migração interrompida a partir do último job bem-sucedido — essencial para repositórios com dezenas ou centenas de jobs SAS.

## Contexto
Sem checkpointing, uma falha no job 60 de 100 obriga re-processar os 59 anteriores (custoso em tokens LLM e tempo). Com `migration_state.json`, o engine registra status por job em tempo real e o `--resume` pula jobs já concluídos.

## Escopo
### Incluído
- Criar `sas2dbx/transpile/state.py` com `MigrationStateManager`:
  ```python
  class MigrationStateManager:
      def __init__(self, output_dir: Path): ...
      def get_job_status(self, job_id: str) -> JobStatus: ...
      def mark_started(self, job_id: str): ...
      def mark_done(self, job_id: str, result: MigrationResult): ...
      def mark_failed(self, job_id: str, error: str): ...
      def get_pending_jobs(self, all_jobs: list[str]) -> list[str]: ...
  ```
- `JobStatus` enum: `PENDING | IN_PROGRESS | DONE | FAILED`
- `migration_state.json` gravado em `{output_dir}/.sas2dbx_state.json`:
  ```json
  {
    "version": "1.0",
    "started_at": "2026-03-30T10:00:00Z",
    "jobs": {
      "job_001_carga_clientes": {"status": "done", "completed_at": "...", "confidence": 0.92},
      "job_002_transform_vendas": {"status": "failed", "error": "LLM timeout"},
      "job_003_report_mensal": {"status": "pending"}
    }
  }
  ```
- Integrar `MigrationStateManager` no `engine.py`:
  - Registrar status antes e após cada job
  - Com `--resume`: chamar `get_pending_jobs()` para pular DONE
  - Sem `--resume`: inicializar estado do zero (sobrescreve state anterior)
- Flag `--resume` já declarada como stub na Story 1.1 — implementar aqui
- Comando CLI `sas2dbx status` para exibir estado atual de uma migração

### Excluído
- Retry automático de jobs FAILED (apenas skip com log)
- Persistência em banco de dados
