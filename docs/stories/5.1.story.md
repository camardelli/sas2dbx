# Story 5.1: schemas.yaml — Migração para list[dict] com tipos

**Status:** Draft → InProgress
**Sprint:** 5 — Inventory Foundation
**Depende de:** nenhuma
**Bloqueada por:** nenhuma

---

## Contexto

O formato atual de `schemas.yaml` usa `list[str]` (apenas nomes de colunas). A Story 5.2 (Source Inventory Module) precisa armazenar tipos de coluna (`STRING`, `DOUBLE`, `DATE`, etc.) capturados via UC REST API. Migrar agora, enquanto o número de consumidores é pequeno e todos são internos, evita dívida técnica futura.

## Objetivo

Migrar o formato interno de schemas de `dict[str, list[str]]` para `dict[str, list[dict]]` onde cada dict tem `{"name": str, "type": str}`. Backward compatibility no reader: schemas.yaml com colunas em formato string antigo são convertidos automaticamente para `{"name": col, "type": "STRING"}`.

## Acceptance Criteria

- [ ] `load_table_schemas()` retorna `dict[str, list[dict]]`
- [ ] Backward compatible: colunas em formato `list[str]` antigas são convertidas para `{"name": col, "type": "STRING"}`
- [ ] `format_schema_context()` usa `col["name"]` — prompt não quebra
- [ ] `enrich_schemas_from_errors()` mantém `list[dict]` internamente
- [ ] `extract_macro_resolutions()` usa `col["name"]`
- [ ] `_schema_context_for_block()` em engine.py funciona com novo formato
- [ ] `_reconcile_columns_with_schema()` e helpers em static_validator.py iteram sobre dicts
- [ ] `_save_schemas()` em preflight.py grava `list[dict]`
- [ ] `_fix_placeholder_add_column()` em fixer.py usa `col["name"]`
- [ ] Todos os testes passam (`pytest` 100% verde)
- [ ] Nenhum schemas.yaml existente em fixtures quebra
