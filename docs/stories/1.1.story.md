# Story 1.1: CLI Foundation — Typer + Rich Progress

**Status:** ToDo
**Prioridade:** P1
**Esforço:** S
**Responsável:** @dev

**Origem:** Revisão arquitetural @architect (R7, R8)

## Objetivo
Configurar o CLI (`sas2dbx/cli.py`) com Typer (em vez de Click) e integrar `rich` para progress bars e output formatado — estabelecendo a base ergonômica para todos os comandos do pipeline.

## Contexto
Typer oferece melhor integração com type hints Python 3.11+ e geração automática de help. `rich` fornece progress bars, tabelas e output colorido essenciais para feedback em migrações de lotes grandes. Definir o CLI correto no Sprint 1 evita migração custosa depois.

## Escopo
### Incluído
- Setup do `sas2dbx/cli.py` com Typer como framework CLI
- App principal com subcomandos: `migrate`, `knowledge`, `validate`
- Integração com `rich` para:
  - Progress bar em `migrate` (jobs processados / total)
  - Tabelas formatadas no relatório de migração
  - Output colorizado por severidade (warnings em amarelo, errors em vermelho)
- Flags globais:
  - `--verbose` / `-v`: output detalhado
  - `--quiet` / `-q`: apenas erros
  - `--config`: path para `sas2dbx.yaml`
  - `--resume`: retomar migração interrompida (integra com Story 3.1)
- Subcomando `knowledge` com: `harvest`, `process`, `build-mappings`, `validate`
- Testes de smoke dos comandos principais

### Excluído
- Implementação da lógica de migração (apenas estrutura do CLI)
- `--resume` funcional (stub que imprime "em breve" — implementação na Story 3.1)
