# Story 0.3: Knowledge Store — Seed dos Mappings Curated

**Status:** Done
**Prioridade:** P0 (bloqueante para Sprint 3 — transpilação)
**Esforço:** M
**Responsável:** @dev
**Epic:** Sprint 0 — Knowledge Store Foundation
**Origem:** Code Review Sprint 0 — G1

## Objetivo
Popular `knowledge/mappings/curated/` com os mapeamentos iniciais de funções SAS, PROCs, dialeto SQL, formatos, informats e dataset options, de forma que `KnowledgeStore.lookup_function()` retorne dados reais para as 50+ funções SAS mais frequentes.

## Contexto
Após a implementação das Stories 0.1 e 0.2, o Knowledge Store estava estruturalmente correto mas vazio — `lookup_function("INTCK")` retornava `None`. Sem seed dos YAMLs curated, o transpiler do Sprint 3 não tem base de conhecimento para as funções mais comuns.

## Escopo
### Incluído
- `knowledge/mappings/curated/functions_map.yaml` — 58 funções SAS → PySpark (String, Numérica, Data/Hora, Condicional/Missing, Conversão)
- `knowledge/mappings/curated/proc_map.yaml` — 10 PROCs com approach (rule/llm/manual) e notas
- `knowledge/mappings/curated/sql_dialect_map.yaml` — 15 regras de diferença SAS SQL → Spark SQL
- `knowledge/mappings/curated/formats_map.yaml` — 20 formatos SAS → PySpark date_format patterns
- `knowledge/mappings/curated/informats_map.yaml` — 10 informats SAS → PySpark to_date patterns
- `knowledge/mappings/curated/options_map.yaml` — 7 dataset options SAS → PySpark equivalentes
- Execução de `build_mappings()` para gerar `mappings/merged/` a partir dos curated/

### Excluído
- Documentação .md em sas_reference/ e pyspark_reference/ (Sprint futuro)
- Harvesting automático de docs HTML/PDF (Stories 0.x futuras)
